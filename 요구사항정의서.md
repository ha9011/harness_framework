# 요구사항 정의서: 영어 패턴 학습기

## 1. 기능 요구사항

### FR-01. 단어 등록

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-01-01 | 단어(word)와 뜻(meaning)을 직접 입력하여 단건 등록할 수 있다 | 필수 |
| FR-01-02 | JSON 형식으로 여러 단어를 한번에 벌크 등록할 수 있다 | 필수 |
| FR-01-03 | 벌크 등록 시 텍스트 영역에 JSON 플레이스홀더 예시가 표시된다 | 필수 |
| FR-01-04 | 이미지(유튜브 캡쳐 등)를 업로드하면 Gemini Vision으로 단어를 추출한다 | 필수 |
| FR-01-05 | 등록 시 Gemini가 품사/발음/유의어/한줄팁을 자동 보강한다 | 필수 |
| FR-01-06 | 벌크 등록은 1회 Gemini 호출로 일괄 보강한다 | 필수 |
| FR-01-07 | 보강 결과를 사용자가 확인/수정한 후 저장한다 | 필수 |
| FR-01-08 | 벌크 등록 시 보강 실패 항목은 보강 없이 저장한다 (부분 성공) | 필수 |
| FR-01-09 | 중복 단어는 알림 후 스킵한다 | 필수 |
| FR-01-10 | 단어에 중요 체크(⭐)를 토글할 수 있다 | 필수 |
| FR-01-11 | 단어를 soft delete할 수 있다 (예문은 유지, 해당 단어의 review_items만 soft delete) | 필수 |
| FR-01-12 | 단어 목록에서 검색(단어/뜻), 품사별 필터, 중요체크만 보기, 등록일 정렬이 가능하다 | 필수 |
| FR-01-13 | 단어 상세에서 해당 단어가 사용된 예문 목록을 볼 수 있다 | 필수 |
| FR-01-14 | 업로드된 이미지는 추출 후 저장하지 않는다 | 필수 |
| FR-01-15 | 등록 시 해당 날짜의 학습 기록에 자동 추가된다 | 필수 |

**JSON 벌크 등록 형식:**
```json
[
  { "word": "drink coffee", "meaning": "커피를 마시다" },
  { "word": "make a bed", "meaning": "침대를 정리하다" }
]
```

### FR-04. 복습 (간격 반복)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-04-01 | 하루 복습 개수를 설정할 수 있다 (10/20/30, DB 저장) | 필수 |
| FR-04-02 | 오늘 복습할 카드를 자동 선정한다 (next_review_date <= 오늘) | 필수 |
| FR-04-03 | 선정 우선순위: ① 모름(HARD) 응답 ② 오래 안 본 것 ③ 복습 횟수 적은 것 | 필수 |
| FR-04-04 | 복습 화면은 [단어][패턴][문장] 탭으로 분리되어 원하는 타입만 골라서 복습할 수 있다. 각 탭에서 N개씩 독립 선정 (설정값 20이면 탭당 최대 20장. 부족한 타입은 있는 만큼만) | 필수 |
| FR-04-05 | 각 탭 내에서 랜덤 셔플하여 표시한다 | 필수 |
| FR-04-15 | 복습 완료 후 [처음부터 다시] 버튼으로 같은 카드를 읽기 전용으로 다시 볼 수 있다 (SM-2 미적용, 응답 버튼 없음) | 필수 |
| FR-04-16 | 복습 완료 후 [추가 복습] 버튼으로 이미 복습한 카드를 제외한 다음 N개를 가져올 수 있다. 더 이상 없으면 "복습할 카드가 없습니다" 표시 | 필수 |
| FR-04-06 | 카드는 양방향이다 (인식: 영→한 / 회상: 한→영) | 필수 |
| FR-04-07 | WORD 인식 카드: 앞면=영어 단어, 뒤면=뜻+예문(있으면)+팁 | 필수 |
| FR-04-08 | WORD 회상 카드: 앞면=한국어 뜻, 뒤면=영어 단어+발음+팁 | 필수 |
| FR-04-09 | PATTERN 인식 카드: 앞면=영어 패턴, 뒤면=한국어 설명+교재 예문 | 필수 |
| FR-04-10 | PATTERN 회상 카드: 앞면=한국어 설명, 뒤면=영어 패턴+교재 예문 | 필수 |
| FR-04-11 | SENTENCE 카드 (RECOGNITION만): 앞면=영어 예문, 뒤면=한국어 해석+상황 | 필수 |
| FR-04-12 | 기억남/애매/모름 응답 후 SM-2 알고리즘으로 간격 자동 조정 | 필수 |
| FR-04-13 | 단어/패턴/예문 등록 시 review_items 자동 생성 (WORD/PATTERN: 양방향, SENTENCE: RECOGNITION만) | 필수 |
| FR-04-14 | 동일 항목 중복 생성 방지 (UNIQUE 제약) | 필수 |

**SM-2 기반 커스텀 간격 조정 규칙:**
- 기억남(EASY): interval × ease_factor × 1.3, ease_factor += 0.15
- 애매(MEDIUM): interval × ease_factor, ease_factor 유지
- 모름(HARD): interval = 1 (리셋), ease_factor = max(1.3, ease_factor - 0.2)
- 초기값: interval_days = 1, ease_factor = 2.5

| FR-08-11 | 홈 화면에 "닉네임님 안녕하세요"를 표시한다 | 필수 |
| FR-08-12 | 모든 학습 데이터(단어, 패턴, 예문, 복습, 학습기록, 설정)가 사용자별 독립 관리된다 | 필수 |
| FR-08-13 | 다른 사용자의 리소스에 접근 시 403을 반환한다 (IDOR 방어) | 필수 |

| FR-08-15 | JWT 만료 (24시간) 시 프론트엔드가 자동 로그아웃 + 로그인 페이지로 리다이렉트한다 | 필수 |

## 2. 비기능 요구사항

| ID | 요구사항 | 설명 |
|----|---------|------|
| NFR-01 | soft delete | 단어/패턴/예문은 물리 삭제 대신 deleted 플래그 사용 |
| NFR-02 | 삭제 연쇄 | 단어/패턴 삭제 시 예문은 유지, 해당 항목의 review_items만 soft delete |
| NFR-03 | TDD | 테스트를 먼저 작성하고 구현 (서비스, 컨트롤러) |
| NFR-04 | 이미지 미저장 | 업로드 이미지는 AI 추출 후 저장하지 않음 |
| NFR-05 | 중복 방지 | 앱 레벨 검증 (DB UNIQUE 제거 — soft delete 재등록 호환). 사용자별 독립 검증 |
| NFR-06 | 인증/인가 | Spring Security + JWT(HttpOnly Cookie). IDOR 방어 (모든 리소스 접근 시 userId 검증) |
| NFR-07 | CORS | allowCredentials(true) + 정확한 origin 지정. 와일드카드 금지 |
