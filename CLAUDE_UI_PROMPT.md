# Claude UI/UX 디자인 요청 프롬프트

너는 사용자 경험(UX)을 최우선으로 생각하는 시니어 UI/UX 디자이너이자, Next.js와 Tailwind CSS에 능숙한 프론트엔드 개발자야. 

내가 기획한 **'영어 패턴 학습기(AI 예문 생성 및 간격 반복 복습)'** 웹 서비스의 핵심 화면 디자인과 UI 컴포넌트 구조를 설계해 주길 바래.

## 1. 프로젝트 핵심 컨셉
- 단어와 패턴을 등록하면, Gemini AI가 난이도별 예문과 '상황(Situation)'을 자동 생성함.
- 교재 사진이나 유튜브 캡쳐를 올리면 AI가 텍스트를 자동 추출해 줌.
- SM-2 기반 간격 반복 알고리즘을 활용해 매일 플립 카드(앞/뒷면) 형태로 복습함.
- **복습은 [단어][패턴][문장] 탭으로 분리**되어 원하는 타입만 골라서 복습 가능.
- 난이도는 **유아/초등/중등/고등** 4단계 (실생활 일상 대화 기준).

## 2. 디자인 요구사항 및 테마 (Design Principles & Theme)
- **Theme: Cozy Cafe & Coffee Tree (코지 카페 & 커피나무 성장)** 
  - 영어 공부를 여유로운 티타임처럼 느낄 수 있는 따뜻하고 아늑한 카페 분위기.
  - **핵심 요소:** 매일 복습을 완료할 때마다 '커피나무'가 조금씩 자라나고 원두를 맺는 게이미피케이션(Gamification) 요소 결합. 복습 연속일(streak)과 연동.
- **Color Palette**: 크림 화이트(배경), 오트밀, 라떼 브라운(포인트), 세이지 그린(식물 요소), 모카 브라운(텍스트). 눈이 편안하고 따뜻한 느낌.
- **Mobile-First**: 출퇴근길 지하철에서 한 손으로 복습하기 편한 모바일 중심의 반응형 UI.
- **Micro-interactions**: 복습 카드가 뒤집어지는(Flip) 부드러운 애니메이션. 버튼이나 카드 모서리는 둥글고 푹신한 느낌(Soft Shadow) 처리.
- **Tailwind CSS**: 모든 디자인 제안은 Tailwind CSS 유틸리티 클래스로 바로 구현할 수 있도록 구체적으로 제시할 것.

## 3. 설계가 필요한 핵심 화면 (총 9개 페이지)

---

### 화면 1: 홈 대시보드 (`/`)
**API:** `GET /api/dashboard`

- **상단**: 오늘의 복습 남은 개수 — **타입별로 분리 표시**
  - 단어: 15장 남음
  - 패턴: 8장 남음 (부족하면 있는 만큼만)
  - 문장: 20장 남음
- **중앙 비주얼**: 나의 커피나무 정원
  - 30일 주기로 하나의 나무가 완성(씨앗→새싹→나무→원두 수확) → **새 화분 추가**
  - 화분들은 **살짝 겹치면서 원근감** 있게 배치 (뒤에 완성된 나무는 작게, 앞에 현재 성장 중인 나무가 크게)
  - 카페 창가에 화분이 줄지어 놓인 분위기
  - 연속일 끊기면 현재 나무 시들기 시작 (다시 복습하면 회복)
- **메인 액션**: 시선을 사로잡는 큰 [오늘의 복습 시작] 버튼
- **하단**: 누적 학습 통계 (단어 N개, 패턴 N개, 예문 N개, 🔥 연속 N일)
- **최근 학습 목록**: Day N - 패턴명 (최대 5건)

---

### 화면 2: 복습 플립 카드 (`/review`)
**API:** `GET /api/reviews/today?type=WORD`, `POST /api/reviews/{id}`

- **탭 인터페이스**: [단어] [패턴] [문장] — 각 탭마다 독립적인 카드 덱 + 진행률
- **카드 영역**: 화면 중앙을 크게 차지하는 카드 UI (카페의 메뉴판이나 냅킨을 연상시키는 디테일 환영)
- **카드 타입별 앞/뒤**:

| 탭 | 카드 종류 | 앞면 | 뒷면 |
|----|----------|------|------|
| 단어 | 인식 (영→한) | 영어 단어 | 뜻 + 예문 2~3개 + 팁 |
| 단어 | 회상 (한→영) | 한국어 뜻 | 영어 단어 + 발음 + 팁 |
| 패턴 | 인식 (영→한) | 영어 패턴 | 한국어 설명 + 교재 예문 |
| 패턴 | 회상 (한→영) | 한국어 설명 | 영어 패턴 + 교재 예문 |
| 문장 | 인식만 | 영어 예문 + 🎭상황(구름 말풍선) | 한국어 해석 |

- **탭(Tap)** 하면 부드럽게 뒤집히면서 뒷면 표시
- **카드 하단 액션 버튼**: [😰 모름(Hard)] [🤔 애매(Medium)] [😊 기억남(Easy)] — SM-2 간격 조정
  - 물뿌리개(기억남: 나무에 물 주기), 구름(애매), 시든 잎(모름) 같은 은유적 아이콘 활용 제안 환영
- **진행률**: 카드 3/20 형태의 카운터
- **복습 완료 후 화면**:
  - 🎉 "단어 복습 완료!" 메시지
  - [🔄 처음부터 다시] 버튼 — 읽기 전용 모드 (SM-2 미적용, [다음 카드 →]만)
  - [➕ 추가 복습하기] 버튼 — 오늘 남은 카드에서 다음 N개 가져옴
  - 더 이상 없으면 "복습할 카드가 없습니다" 빈 상태

---

### 화면 3: 단어 목록 + 등록 (`/words`)
**API:** `GET /api/words`, `POST /api/words`, `POST /api/words/bulk`, `POST /api/words/extract`

- **등록 영역** (3가지 방식 탭 또는 토글):
  - 단건 입력: word + meaning 텍스트 필드
  - JSON 벌크: 플레이스홀더 예시 포함 textarea
  - 이미지 업로드: 점선 테두리 Dropzone → AI 추출 중 로딩 (커피잔에 커피가 채워지는 애니메이션 등) → 추출 결과를 편집 가능한 폼으로 표시 → [저장]
- **벌크 등록 결과 UI**: saved(성공) / skipped(중복) / enrichmentFailed(보강 실패) 3가지 상태를 시각적으로 구분 표시
- **검색/필터 바**: 검색어 입력 + 품사 드롭다운 + [⭐ 중요만] 체크 + 정렬(등록일/알파벳)
- **단어 목록**: 페이지네이션, 각 단어마다 ⭐ 토글 버튼

---

### 화면 4: 단어 상세 (`/words/[id]`)
**API:** `GET /api/words/{id}`, `POST /api/generate/word`

- 단어 정보 카드: 단어, 뜻, 품사, 발음, 유의어, 한줄 팁
- 이 단어가 사용된 예문 목록
- [예문 추가] 버튼 → 난이도(유아/초등/중등/고등) + 개수(5/10개) 선택 → 생성

---

### 화면 5: 패턴 목록 + 등록 (`/patterns`)
**API:** `GET /api/patterns`, `POST /api/patterns`, `POST /api/patterns/extract`

- **등록 영역** (2가지):
  - 직접 입력: template + description + 예문(동적 추가)
  - 이미지 업로드: Dropzone → AI 추출 → 확인/수정 폼 → [저장]
- **패턴 목록**: 페이지네이션, 각 패턴의 template + 교재 예문 미리보기

---

### 화면 6: 패턴 상세 (`/patterns/[id]`)
**API:** `GET /api/patterns/{id}`, `POST /api/generate/pattern`

- 패턴 정보: template, description
- 교재 예문 목록 (순서 보존, 1~10번)
- 이 패턴으로 생성된 예문 목록 (화면 7과 동일한 아코디언 형태: 영어+상황 표시, 탭하면 해석 펼쳐짐)
- **[이 패턴으로 예문 생성]** 버튼 → 난이도(유아/초등/중등/고등) + 개수(10/20/30) 선택 → 생성

---

### 화면 7: 예문 생성 (`/generate`)
**API:** `POST /api/generate`

- **조건 선택**:
  - 난이도: [유아] [초등] [중등] [고등] 라디오/칩 (각 레벨 느낌 표시: "카페 주문 수준", "친구랑 카톡 수준" 등)
  - 개수: [10] [20] [30]
- **로딩 UI**: Gemini API 대기 중 (10~30초 걸릴 수 있음) — 커피가 내려지는 애니메이션 등
- **결과 표시**: 예문 리스트
  - 각 예문: 영어 문장 + 🎭상황 표시
  - [탭하면 해석 펼쳐짐] → 한국어 해석 + 📌패턴 태그 + 📌단어 태그

---

### 화면 8: 학습 기록 (`/history`)
**API:** `GET /api/study-records`

- 날짜별 역순 목록
- 각 날짜: "Day N" + 등록한 패턴 + 단어 리스트
- 페이지네이션

---

### 화면 9: 설정 (`/settings`)
**API:** `GET /api/settings`, `PUT /api/settings/{key}`

- 하루 복습 개수: [10] [20] [30] 라디오 선택 (타입별 각 N개씩 = 총 3N장)
- [저장] 버튼

---

## 4. 공통 UI 요소

### 하단 네비게이션 바 (Mobile)
모바일 하단에 고정된 네비게이션:
- [🏠 홈] [📖 단어] [🔤 패턴] [✨ 생성] [🃏 복습]
- 현재 위치 하이라이트
- 학습 기록, 설정은 홈 화면 내 링크 또는 상단 메뉴로 접근

### 빈 상태 (Empty State) 디자인
첫 사용자가 데이터 없이 들어왔을 때 각 화면에서:
- 홈: 커피나무 씨앗 + "첫 단어를 등록하고 나무를 심어보세요!" + [단어 등록하러 가기] 버튼
- 복습: "아직 복습할 카드가 없어요. 단어나 패턴을 먼저 등록해보세요!"
- 예문 생성: "등록된 단어/패턴이 필요해요" + [등록하러 가기]
- 전체적으로 따뜻한 안내 문구 + 다음 행동 유도 버튼

### 삭제/수정 UI
- 단어/패턴 목록: 스와이프 또는 롱프레스로 [수정][삭제] 액션 표시
- 단어/패턴 상세: 우측 상단 [...] 메뉴 → [수정] [삭제]
- 삭제 시 확인 모달: "정말 삭제하시겠어요?" (soft delete이므로 복구 가능하다는 안내)

### SENTENCE 카드 — 상황 표시 디자인
- 상황(situation)은 영어 예문과 **같은 앞면에 함께 표시**
- **구름 말풍선 UI**로 카드 상단 또는 예문 위에 떠있는 느낌
- 부드러운 세이지 그린 배경 + 둥근 모서리 + 작은 꼬리(말풍선 꼬리)
- 예문을 읽기 전에 "이 상황을 상상하며 읽어보세요" 느낌을 주는 시각적 분리

### 대시보드 [복습 시작] 동작
- [복습 시작] 누르면 `/review` 페이지로 이동, **[단어] 탭이 기본 선택**
- 각 탭에 남은 개수 뱃지 표시 (예: [단어 15] [패턴 8] [문장 20])

---

## 5. 요청 사항
위 9가지 화면 + 공통 UI 요소에 대해 다음 항목들을 작성해 줘.
1. **전체적인 컬러 팔레트 (Hex 코드 포함)와 타이포그래피 제안**
2. **각 화면의 모바일 UI 레이아웃 구조 (상하 배치, 여백 등)**
3. **주요 UI 요소에 대한 Tailwind CSS 클래스 예시** (예: 복습 카드의 섀도우, 둥근 모서리, 플립 애니메이션 클래스 등)
4. **'커피나무 성장' 테마를 UI에 자연스럽게 녹여내기 위한 디테일한 아이디어 2~3가지**
5. **복습 완료 시 축하 인터랙션** (커피나무 물 주기, 원두 수확 등 어떻게 표현할지)
