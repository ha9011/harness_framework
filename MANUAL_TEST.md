# 수동 테스트 케이스

> 순서대로 진행. 각 항목 체크(✅/❌) 후 비고 기록.

## 사전 조건

- [ ] Docker Desktop 실행 중
- [ ] `docker compose up -d` → PostgreSQL 기동 확인
- [ ] `cd backend && ./gradlew bootRun --args='--spring.profiles.active=local'` → 8080 기동
- [ ] `cd frontend && npm run dev` → 3000 기동
- [ ] 브라우저에서 http://localhost:3000 접속

## 비고

- 테스트 순서가 중요: 2단계에서 등록한 데이터를 이후 단계에서 사용
- 2단계에서 벌크 중복 포함 테스트(2.10)로 단어가 4개(apple, banana, cherry, dog)가 됨
- 5단계(복습)는 등록 직후 next_review_date=오늘이므로 바로 테스트 가능
- 8단계 삭제 후에는 데이터가 줄어들므로 마지막에 진행
- 10단계(SM-2 간격)는 다음 날 진행
